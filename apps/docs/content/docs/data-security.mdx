# Data Security

Windback handles your customers' personal information (emails, names) — we take protecting it seriously.

## Encryption at Rest

All customer PII (email addresses and names) is encrypted before it is stored in our database. If someone gains access to the raw database, they see only encrypted blobs — not your customers' data.

Your dashboard and API responses show decrypted values as normal, behind authentication.

## PII Masking in Logs

Customer emails never appear in plain text in application logs. They are automatically masked:

```
sarah@acme.co  ->  sa***@acme.co
```

This prevents accidental exposure through log aggregation services.

## Encryption in Transit

All API traffic uses HTTPS (TLS 1.2+). Webhook payloads to your endpoints are signed so you can verify authenticity.

## Infrastructure

- Database connections use SSL in production
- Authentication tokens are signed and expire after 7 days
- API keys are hashed before storage — we never store the raw key
- Rate limiting protects against abuse
- CORS is restricted to your configured domains
