# Churn Risk Prediction

Windback tracks customer behavioral events (logins, feature usage, billing portal visits, etc.) and calculates a churn risk score for every customer. When a score crosses your threshold, an AI-generated retention email is sent automatically — either directly by Windback or via webhook to your system.

## How It Works

1. **You send behavioral events** — Use the [Tracking API](/docs/api-reference/tracking) or the JavaScript snippet to send events like `login`, `feature_used`, `plan_downgrade`, etc.
2. **Windback calculates risk scores** — Every 24 hours (or on-demand), Windback analyzes the last 14 days of events per customer and scores them 0–100.
3. **AI generates retention emails** — When a customer crosses the auto-email threshold, an AI writes a personalized retention email based on the detected signals.
4. **Email is delivered** — Either Windback sends it directly, or it POSTs the content to [your webhook](/docs/churn-risk-webhook) for you to send.

## Churn Signals

Windback detects both negative (risk-increasing) and positive (risk-decreasing) signals:

### Negative Signals

| Signal | Default Weight | Trigger |
|--------|---------------|---------|
| `gone_dark` | 35 | No login in 14+ days |
| `login_frequency_drop` | 25 | Login frequency dropped 50%+ week-over-week |
| `core_feature_unused` | 20 | Core feature not used in 7 days |
| `payment_failed` | 20 | Active payment failure |
| `billing_portal_visit` | 15 | Visited billing/pricing page |
| `team_member_removed` | 15 | Removed a team member |
| `plan_downgrade` | 15 | Downgraded plan |
| `support_spike` | 10 | 3+ support tickets in 7 days |
| `data_export` | 10 | Exported data |
| `cancel_button_clicked` | 10 | Opened cancel widget |
| `session_duration_drop` | 10 | Session duration dropped 60%+ |
| `api_volume_drop` | 8 | API call volume dropped 60%+ |
| `feature_breadth_shrink` | 8 | Using fewer distinct features |

### Positive Signals

| Signal | Default Weight | Trigger |
|--------|---------------|---------|
| `plan_upgrade` | -20 | Upgraded plan |
| `invite_sent` | -15 | Invited new team member |
| `new_feature_adopted` | -10 | Used a new feature |
| `api_volume_growing` | -10 | API usage increasing |
| `login_streak` | -8 | Logged in every day this week |

Scores are clamped to 0–100. Per-project weight overrides are supported via the config API.

## Risk Levels

| Level | Default Threshold |
|-------|------------------|
| High | Score >= 70 |
| Medium | Score >= 40 |
| Low | Score < 40 |

Thresholds are configurable per project in **Settings > Churn Risk**.

## Auto-Email Delivery

When auto-email is enabled and a customer's score crosses the threshold, Windback generates a personalized retention email. There are two delivery modes:

### Direct Email (Default)

Windback sends the email directly to the customer using your configured email method (Windback default, Gmail OAuth, or custom domain).

### Webhook Delivery (Recommended for Privacy)

Instead of sending emails directly, Windback POSTs the AI-generated email content to your webhook endpoint. Your system maps the `customer_id` to the real email address and sends it using your own email infrastructure.

This means **Windback never needs real customer email addresses** — you can track events using opaque identifiers like `usr_abc123`.

See the full [Webhook Delivery Guide](/docs/churn-risk-webhook) for setup instructions, payload format, and code examples.

## Configuration API

### Get Config

```
GET /api/v1/projects/:slug/churn-risk/config
```

### Update Config

```
PUT /api/v1/projects/:slug/churn-risk/config
```

```json
{
  "enabled": true,
  "high_risk_threshold": 70,
  "medium_risk_threshold": 40,
  "auto_email_enabled": true,
  "auto_email_threshold": 70,
  "webhook_url": "https://your-app.com/api/windback-webhook"
}
```

When you set `webhook_url`, a `webhook_secret` is auto-generated and returned in the response. Use this secret to verify payloads.

### Recalculate Scores

```
POST /api/v1/projects/:slug/churn-risk/recalculate
```

Triggers an immediate recalculation for all customers in the project.

## Dashboard

Navigate to **Churn Risk** in the sidebar to view:

- **Stats cards** — Total customers, high/medium/low risk counts, emails sent
- **Risk scores table** — Sortable by score, filterable by risk level
- **Trend indicators** — Whether each customer's risk is increasing, stable, or decreasing
